# Проект для сети электроники (Пакет +).  
Проект представляет собой систему для управления сетью магазинов электроники, предоставляющую возможности по централизованному контролю магазинов, сотрудникам, товарному ассортименту и продажам. 
### Пакет +: 
В проекте реализована одна асинхронная задача (ежедневный анализ продаж и отправка отчёта админам).  
## Назначение
Данный проект создан для эффективного управления организацией розничной сети электроники. Он обеспечивает удобный API для создания, чтения, обновления и удаления данных о магазинах, сотрудниках, товарах и продажах, а также реализует систему прав доступа и фильтрации данных.
## Функционал
* CRUD операции для моделей: Store (магазин), Employee (сотрудник), Product (товар), Sale (продажа).
* Аутентификация и авторизация через JWT.
* Кастомные права доступа: только активные сотрудники могут работать с API магазинов.
* Настроенный фильтр по стране и поиск по ключевым полям.
* Запрет обновления отдельных полей (например, задолженности по поставщику).
* Встроенный интерфейс администратора Django с дополнительными функциональными возможностями.
* Покрытие основных сценариев API функциональными тестами.
* Асинхронная очередь задач Celery и интеграция с Redis. (Одна задача в подарок)
## Установка и запуск  
### Клонируйте репозиторий:
git clone https://github.com/carplatinum/el_network  
cd el_network  
### Создайте и активируйте виртуальное окружение:
python -m venv .venv  
source .venv/bin/activate   # Linux/macOS  
.\.venv\Scripts\activate    # Windows PowerShell  
### Установите зависимости:
pip install -r requirements.txt  
Настройте переменные окружения, укажите параметры базы данных, секретный ключ и прочие необходимые параметры.
### Выполните миграции базы данных:
python manage.py migrate  
### Создайте суперпользователя для доступа к админ-панели:
python manage.py createsuperuser  
### Запустите сервер разработки:  
python manage.py runserver  
### Запустите Celery worker:  
- На Linux/macOS/WSL:  
  celery -A config worker -l INFO
- В Windows PowerShell:  
  celery -A config worker -l INFO -P solo  
### Для запуска тестов выполните:  
pytest  
## Используемые технологии  
* Python 3.8+
* Django 5.2
* Django REST Framework 3.16
* PostgreSQL (через psycopg2)
* Celery + Redis для фоновых задач
* django-filter для расширенной фильтрации API
* JWT (djangorestframework_simplejwt) для авторизации
